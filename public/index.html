ParaScript

A TypeScript dialect that adds the things JavaScript still doesn't have built in: reactive signals, edge-triggered handlers, integer ranges, a pipeline operator, error-chain operators, and compile-time purity.

Files end in .pts. Every extension desugars to standard JavaScript at parse time. Compile with bun build, alias para:* in your bundler to parabun-browser-shims, and the result runs in a browser, on a Lambda, on Cloudflare Workers, on Deno, or on Node.

What you write

signal count = 0;
signal doubled = count * 2;          // derived — re-runs when count changes

effect { console.log(doubled); }     // logs every change

when count >= 10 {                   // edge-triggered: fires once per 0→1
  alert("ten!");
}

count++;                             // bare-read sugar: count.set(count.get() + 1)

What it compiles to

const __s = require("para:signals");
const count = __s.signal(0);
const doubled = __s.derived(() => count.get() * 2);
__s.effect(() => { console.log(doubled.get()); });
__s.onRising(() => count.get() >= 10, () => { alert("ten!"); });
count.set(count.get() + 1);

No virtual DOM, no compiler runtime, no magic. The para:* modules are small, normal JS — see para:signals for the entire reactive engine.

Run it anywhere

Five-line bundler alias is the whole story:

// vite.config.ts
export default defineConfig({
  resolve: {
    alias: [{ find: /^para:(.*)$/, replacement: "parabun-browser-shims/$1" }],
  },
});

Then bun build src/main.pts and bundle as normal. The output is plain JavaScript that ships to a browser, an edge function, a Lambda, or any Node runtime. See the install guide for the esbuild / webpack / rollup variants.

Why not just TypeScript

Because plain TS makes you write the boilerplate by hand every time. signal x = 0 is one line; the equivalent reactive-store wiring across React, Solid, Svelte, MobX, Zustand, Jotai, etc. is whatever ceremony the framework demands, and the syntax never collapses. ParaScript treats the things you write a hundred times a week — reactive bindings, edge-triggered handlers, ranges, pipelines, error chains — as first-class syntax instead of library APIs.

Every extension is opt-in and parse-time. Plain .ts files behave exactly as TypeScript expects; ParaScript only kicks in when you switch the extension to .pts.

Sister project — Parabun

Parabun is a fork of Bun that bundles ParaScript natively (no build step, no bundler alias) plus a stack of native runtime modules — raw CUDA / Metal kernels, V4L2 camera capture, ALSA audio, on-device LLM inference, GPIO / I²C / SPI for Linux SBCs. If you control the server (or you're on a Pi / Jetson / NUC) and you want hardware acceleration in TypeScript, that's the runtime.

ParaScript is the language. Parabun is the runtime that ships it with batteries. Use either, both, or just ParaScript and a normal Node host.
