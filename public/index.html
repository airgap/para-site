ParaScript

A TypeScript dialect that adds the things JavaScript still doesn't have built in: reactive signals, edge-triggered handlers, integer ranges, a pipeline operator, error-chain operators, and compile-time purity. Files end in .pts.

Every extension desugars to standard JavaScript at parse time. There is no ParaScript runtime layer beyond a small npm package the desugarings call into — so the output of a .pts file ships to a browser, an edge function, a Lambda, Deno, or any Node host. No runtime fork required.

What you write

Five things to know on the way in. Each desugars to a normal JS call — the language only adds syntax, not semantics.

Reactive signals

signal count = 0;
signal doubled = count * 2;          // derived — re-runs when count changes

effect { console.log(doubled); }     // logs every change

count++;                             // bare-read sugar: count.set(count.get() + 1)

Edge-triggered handlers

signal score = 0;

when score >= 100 {                  // fires once per false→true transition
  unlockAchievement("century");
}

when not online {                    // fires on the true→false transition
  showOfflineBanner();
}

Reactive bindings (no effect block)

signal name = "world";

name ~> document.title;              // assignment binding: title stays in sync
name -> console.log;                 // call binding: console.log fires on every change

Pipelines & ranges

// Integer ranges
for (const i of 0..n) work(i);
const evens = 0..=20 |> filter(i => i % 2 === 0);

// Pipeline operator threads values left-to-right
const out = pixels |> map(p => p * 1.2) |> clamp(0, 255);

Error chains

// `..!` is .catch, `..&` is .finally, `..=` is await-assign.
const data ..= fetch(url).then(r => r.json()) ..! err => defaults ..& () => spinner.hide();

Run it anywhere

ParaScript compiles to standard JavaScript with a handful of import calls into a normal npm package (parabun-browser-shims, the runtime side of the language). To make those import strings resolve, your bundler needs a five-line alias. That's the whole story. 

// vite.config.ts
import { defineConfig } from "vite";

export default defineConfig({
  resolve: {
    alias: [{ find: /^para:(.*)$/, replacement: "parabun-browser-shims/$1" }],
  },
});

esbuild, webpack, rollup — all the same idea, four lines of plugin or config. See the install guide for the recipes.

For the build step itself you need a transpiler that understands .pts. Today that's bun build — Bun's parser is what knows how to parse the syntax, even if you ship the output to a non-Bun host. A standalone @parascript/transpile npm package (no Bun required) is on the roadmap.

$npm install parabun-browser-shims
$bun build src/main.pts --outdir dist/

Frontend example — live search, no framework

A search input that filters a list reactively. signal for the query, derived (autopromoted from a signal with a tracked RHS) for the filtered list, effect to repaint the DOM. No React, no Vue, no Svelte, no compiler runtime — this ships as ~2 KB of JS plus the shim package. 

// src/search.pts
import items from "./items.json";

signal query = "";
signal results = items.filter(it => it.name.toLowerCase().includes(query.toLowerCase()));

const input = document.querySelector<HTMLInputElement>("#q")!;
const list  = document.querySelector<HTMLUListElement>("#results")!;

input.addEventListener("input", e => query = (e.target as HTMLInputElement).value);

effect {
  list.innerHTML = results.slice(0, 10)
    .map(it => `<li>${it.name}</li>`)
    .join("");
}

when results.length === 0 {
  list.innerHTML = "<li class='empty'>no matches</li>";
}

The signal results = items.filter(...) declaration auto-promotes to a derived() because its RHS reads query. The effect { } repaints whenever results changes; the when block fires once each time results.length transitions to zero, not on every keystroke that keeps it zero.

Backend example — live connection state on a plain Node server

WebSocket server with per-connection signals. Each socket gets its own messages count and idle derived; an edge-triggered when closes the connection after 60s of silence. Runs on plain Node, Bun, Deno, anywhere. 

// src/server.pts
import { WebSocketServer } from "ws";

const wss = new WebSocketServer({ port: 8080 });

wss.on("connection", ws => {
  signal messages = 0;
  signal lastSeen = Date.now();
  signal idle = (Date.now() - lastSeen) > 60_000;

  ws.on("message", () => {
    messages++;
    lastSeen = Date.now();
  });

  // Heartbeat tick refreshes `idle` so the `when` below can fire
  const ping = setInterval(() => lastSeen = lastSeen, 5_000);

  when idle {
    ws.close(1000, "idle timeout");
    clearInterval(ping);
  }

  // Per-connection log line that re-fires on every meaningful state change
  effect { console.log(`conn ${ws.id}: ${messages} msgs, idle=${idle}`); }
});

The Lambda / Workers shape is the same: each request gets its own scope, signals stand in for what would otherwise be a per-request closure or a hand-rolled state object. Cleanup — the setInterval — lives next to the state it manages.

Pipeline example — image processing in the browser

The pipeline operator |> chains typed-array kernels into a single fused pass — one loop, no intermediate arrays. This runs on the canvas ImageData straight from the browser, no WASM and no GPU. 

// src/filter.pts
import { mulScalar, add, clamp } from "para:simd";

const canvas = document.querySelector<HTMLCanvasElement>("#in")!;
const out    = document.querySelector<HTMLCanvasElement>("#out")!;
const ctx = canvas.getContext("2d")!, octx = out.getContext("2d")!;

signal brightness = 1.0;
signal img = ctx.getImageData(0, 0, canvas.width, canvas.height);

document.querySelector<HTMLInputElement>("#bright")!
  .addEventListener("input", e => brightness = +(e.target as HTMLInputElement).value);

effect {
  // Fused: one pass over the pixel buffer, no intermediate Float32Arrays.
  const px = new Float32Array(img.data);
  const adjusted = px |> mulScalar(brightness) |> clamp(0, 255);
  octx.putImageData(new ImageData(new Uint8ClampedArray(adjusted), canvas.width), 0, 0);
}

On Parabun the same pipeline lifts to a single GPU dispatch when the input is large enough; in the browser you stay on CPU but still get the fusion win — one tight loop instead of three.

Error chains — flat .catch / .finally without ceremony

..! is .catch, ..& is .finally, ..= is await-assign. They compose: read the chain top-to-bottom, no nesting, no try/catch indent rabbit hole. 

// API call cascade with a fallback, a cleanup, and an awaited assignment.
const profile ..= fetch(`/api/users/${id}`)
  .then(r => r.json())
  ..! err => ({ id, name: "anonymous", _stub: true })   // fallback on any failure
  ..& () => loading.set(false);                         // always runs

// Without ParaScript:
let profile;
try {
  const r = await fetch(`/api/users/${id}`);
  profile = await r.json();
} catch (err) {
  profile = { id, name: "anonymous", _stub: true };
} finally {
  loading.set(false);
}

Why not just TypeScript

Because plain TS makes you write the boilerplate by hand every time. signal x = 0 is one line; the equivalent reactive-store wiring across React, Solid, Svelte, MobX, Zustand, Jotai, Pinia, whatever-shipped-this-month is whatever ceremony the framework demands — and the syntax never collapses to the same place. ParaScript treats the things you write a hundred times a week as first-class syntax instead of library APIs.

Every extension is opt-in and parse-time. Plain .ts files behave exactly as TypeScript expects — ParaScript only kicks in when you switch the extension to .pts. Your existing codebase doesn't change; new files can adopt the syntax incrementally.

Sister project — Parabun

Parabun is a fork of Bun that bundles ParaScript natively (no build step, no bundler alias) plus a stack of native runtime modules: raw CUDA / Metal kernels, V4L2 camera capture, ALSA audio, on-device LLM inference, GPIO / I²C / SPI for Linux SBCs, statically-linked image / audio / CSV codecs.

ParaScript is the language. Parabun is the runtime that ships it with batteries. Use either, both, or just ParaScript and a plain Node / browser host — the language doesn't care.

Get started

  1. npm install parabun-browser-shims — the runtime side of the language, ~kb-scale, pure JS.
  2. curl -fsSL https://bun.sh/install | bash — Bun for the build step (until the standalone transpiler ships).
  3. Add the bundler alias from the install guide (Vite / esbuild / webpack / rollup).
  4. Rename a .ts file to .pts and start using signal / effect { } / when { }.

Editor support: curl -fsSL https://raw.githubusercontent.com/airgap/parabun/main/install-extension.sh | bash installs the VS Code / Cursor / Kiro extension — .pts grammar plus an LSP with hover, go-to-definition, purity diagnostics, and operator documentation.
